# PulseVoice — Executive Summary

**PulseVoice** is a local-first macOS daemon + menubar app that taps the *structured* output of CLI coding agents and narrates it aloud, content-aware, so agent responses can be consumed eyes-free. It is the output-side complement to voice input (Wispr Flow): the problem it solves is not "having to read" but triaging walls of mixed prose-and-code to find the load-bearing 10%, and the dead time while a long agent task runs. Success is **daily personal use** — reliable enough to step away during a long run and trust being called back when the agent needs input.
**v1 scope (MVP):** Claude Code as the only source (Stop/Notification hooks → JSONL transcript over a Unix socket); a content-aware **chunker** (the crown jewel — role × size tiering that announces operational edits, reads prose, normalizes inline code, never reads verbatim code); a code→speakable **normalization** pass; **local Kokoro TTS** (warm-resident, on-device — a privacy/exfil-driven default, not just cost); a cancellable **playback queue** with barge-in; **mode toggle**, **trivial-turn filter**, **pull-on-demand** (full read-out only in v1), and **always-on attention events** that preempt across all sessions so a blocked permission prompt is never missed. Built in **Rust** (a ~5-crate workspace + a Rust-first menubar shell, Swift as a triggered fallback) behind two trait seams — `SourceAdapter` and `TtsProvider` — that keep everything downstream surface-agnostic.
**Deliberately deferred to Phase 2:** the Codex CLI + desktop adapters, the summarize/re-voice LLM preprocessors, token-streaming, voice control, and cloud premium voices. **Known risks** (all technical): chunker quality, the unversioned Claude Code JSONL contract (mitigated by a schema probe + loud flattened-text degradation), and local-TTS first-audio latency on M1 (the load-bearing unknown to validate early). **Security posture:** single-user, local-only, no auth/no cloud in the default path; secret-redaction is a recorded aware-but-deferred item under an explicit v1 assumption of a private audio environment.
